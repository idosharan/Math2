# פרומפט ליצירת אפליקציית תרגול אינטראקטיבית לילדים

## תבנית פרומפט (העתק והתאם לפי הצורך):

---

צור אפליקציית תרגול אינטראקטיבית לילדים בקובץ HTML יחיד (כולל CSS ו-JavaScript מוטמעים). האפליקציה צריכה להיות בעברית (RTL), מותאמת למובייל ומחשב, ובעלת עיצוב מרהיב בערכת נושא של **[בחר ערכת נושא: גיימינג/חלל/אוקיינוס/ג'ונגל]**.

### עיצוב ומראה
- רקע כהה (#121212) עם אלמנטים ניאוניים (ירוק רעל, כחול חשמל, סגול ורוד).
- כרטיסים כהים עם גבולות זוהרים (Neon glow).
- גופן נקי (Segoe UI / Varela Round).
- מערכת נקודות: **[XP / כוכבות / מטבעות]** ומערכת חיים: **[לבבות / שילדים / חיי בוס]**.
- אנימציות חלקות: כניסת כרטיסים, רעידה (Shake) לתשובה שגויה, פעימה (Pulse) עם אפקט "Level Up" לתשובה נכונה.
- קונפטי בסיום שלב בהצלחה.
- אפקטים קוליים (Web Audio API) — צלילי 8-bit לאיסוף, שגיאה, וניצחון.
- רספונסיבי מלא למובייל ומחשב.

### מבנה מסכים
א. **מסך פתיחה (Welcome Screen):** כותרת לוגו בסגנון ערכת הנושא, כפתור "START GAME" מהבהב, כפתורי עולמות (שלבים) עם אייקונים.

ב. **מסך בחירת תת-שלב (Mission Select):** רשת כפתורים למשימות, כל משימה מציגה שם ו"10 שאלות למעבר שלב", אופציית "Boss Fight" (מיקס קטגוריות).

ג. **מסך שאלה (Battle Screen):** סרגל התקדמות, מדדים (XP, Combo, HP), כרטיס שאלה, תשובות בחירה מרובה (4 אופציות) או הקלדה חופשית, כפתור "רמז".

ד. **שכבת משוב (Feedback Overlay):** טקסטים של משחקים ("COMBO!", "CRITICAL HIT!", "MISS..."), הצגת תשובה נכונה בטעות, מעבר אוטומטי/כפתור המשך.

ה. **מסך תוצאות (Mission Passed):** דירוג (S/A/B/C), סיכום XP, תשובות נכונות, קומבו מקסימלי, כפתורי "Play Again"/"Next Mission"/"Main Menu".

### מבנה נתונים — בנק שאלות
השאלות מאורגנות באובייקט עם מפתחות לפי שלבים. סוגי שאלות: `type: "mc"` (בחירה מרובה) או `"type"` (הקלדה). לפחות 20-30 שאלות מכל סוג לייצור אקראיות.

#### **[התאם את התוכן הלימודי]:**
- **שלב 1:** [נושא בסיסי — למשל: כפל וחילוק עד 10×10]
  - תתי שלבים: [למשל: כפל, חילוק, מיקס]
- **שלב 2:** [נושא מתקדם — למשל: מספרים עד 9,999]
  - תתי שלבים: [למשל: ערכי ספרות, עיגול, כפל וחילוק גדולים]
- **שלב 3:** [נושא מאתגר — למשל: סדר פעולות + בעיות מילוליות]
  - תתי שלבים: [למשל: סדר פעולות, בעיות מילוליות, בוס]

### דגשים טכניים
- קובץ HTML יחיד בלבד, ללא תלויות חיצוניות (no CDN, no frameworks).
- כיוון RTL (`direction: rtl`, `lang="he"`).
- מיקוד אוטומטי (focus) על שדה הקלדה.
- לחיצה על Enter שולחת את התשובה.
- בשאלות הקלדה (`input[type=number]`) לבטל את החצים (spinners).

## מה לשנות בין אפליקציות:
1. **ערכת נושא** — שנה צבעים, אייקונים, וטקסטים של משוב
2. **תוכן לימודי** — עדכן את בנק השאלות לנושא הרצוי
3. **כיתה/רמת קושי** — התאם את הטווחים המתמטיים
4. **שפה** — ניתן להחליף לשפה אחרת (שנה `lang` ו-`dir`)
5. **מספר שלבים** — הוסף/הסר שלבים לפי הצורך
